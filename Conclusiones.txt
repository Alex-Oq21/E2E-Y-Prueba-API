2. CAPA DE API (Karate Framework - Swagger PetStore)
----------------------------------------------------------------------
* Análisis Crítico de Negocio (Posible Bug): 
  Quitando la obviedad de que Swagger PetStore es un entorno público de 
  pruebas, si evaluamos este comportamiento bajo un contexto empresarial 
  real, existe una vulnerabilidad lógica/excepción de negocio no controlada: 
  El endpoint 'POST /pet' permite registrar una mascota con un ID que ya 
  existe en la base de datos con los mismos datos de forma idéntica, 
  sobrescribiendo el recurso de manera silenciosa en lugar de retornar un 
  error de conflicto (ej. 409 Conflict) o una excepción que indique 
  "La mascota ya existe".

* Comportamiento del Flujo: A pesar del hallazgo anterior, el diseño de 
  la API es consistente en su operación técnica, permitiendo que los 
  demás escenarios (Consultar por ID, Actualizar con PUT y Filtrar por 
  estatus) se ejecuten y encadenen correctamente de punta a punta.

* Mantenibilidad del Framework: La arquitectura del proyecto se diseñó 
  aplicando las mejores prácticas, aislando los escenarios para cumplir 
  con la independencia de las pruebas y separando los datos de entrada 
  (Payloads JSON externos) de la lógica del archivo de pruebas (.feature).
